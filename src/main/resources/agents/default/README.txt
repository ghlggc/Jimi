=============================================================================
Default Agent 配置说明
=============================================================================

这是 Jimi 的默认主 Agent，具有完整的工具访问权限和子 Agent 管理能力。

-----------------------------------------------------------------------------
文件说明
-----------------------------------------------------------------------------

agent.yaml              - Agent 配置文件（必需）
system_prompt.md        - 系统提示词（必需）
config-example.yaml     - 完整配置示例（参考）
README.txt              - 本说明文件

-----------------------------------------------------------------------------
配置文件结构
-----------------------------------------------------------------------------

agent.yaml 包含以下部分：

1. 基本信息
   - name: Agent 名称
   - description: Agent 描述

2. 系统提示词
   - system_prompt: 提示词文件路径
   - system_prompt_args: 自定义参数（可选）

3. 工具列表
   - tools: 可用工具数组

4. 子 Agent
   - subagents: 子 Agent 配置映射

-----------------------------------------------------------------------------
可用工具
-----------------------------------------------------------------------------

文件操作：
  ReadFile         - 读取文件
  WriteFile        - 写入文件
  StrReplaceFile   - 字符串替换
  PatchFile        - 补丁修改
  Glob             - 文件搜索
  Grep             - 内容搜索

Shell：
  Bash             - 执行命令

Web：
  SearchWeb        - 网页搜索
  FetchURL         - 获取内容

辅助：
  Think            - 思考推理
  Todo             - 任务管理

自动注册：
  Task             - 子 Agent 委托（有 subagents 时自动添加）

-----------------------------------------------------------------------------
子 Agent 说明
-----------------------------------------------------------------------------

Build Agent (build)
  职责：构建项目、修复编译错误、管理依赖
  触发：涉及编译、构建、依赖相关任务

Test Agent (test)
  职责：运行测试、分析失败、提高覆盖率
  触发：涉及测试运行、测试编写、覆盖率检查

Debug Agent (debug)
  职责：调试错误、修复 bug、分析崩溃
  触发：涉及运行时错误、异常、调试

Research Agent (research)
  职责：技术调研、查找最佳实践、对比方案
  触发：涉及学习、调研、查找文档

-----------------------------------------------------------------------------
修改配置
-----------------------------------------------------------------------------

1. 修改工具列表
   在 agent.yaml 的 tools 数组中添加/删除工具名称

2. 修改子 Agent
   在 agent.yaml 的 subagents 中添加/删除子 Agent 配置

3. 修改系统提示词
   编辑 system_prompt.md 文件

4. 添加自定义参数
   在 agent.yaml 的 system_prompt_args 中添加键值对

-----------------------------------------------------------------------------
示例：添加自定义子 Agent
-----------------------------------------------------------------------------

1. 创建子 Agent 目录和配置
   agents/custom/agent.yaml
   agents/custom/system_prompt.md

2. 在 agent.yaml 中注册
   subagents:
     custom:
       path: agents/custom/agent.yaml
       description: Custom specialized agent

3. 重启 Jimi 使配置生效

-----------------------------------------------------------------------------
注意事项
-----------------------------------------------------------------------------

1. 子 Agent 会自动触发 Task 工具注册
2. 修改配置后需重启 Jimi
3. 系统提示词支持变量：{{KIMI_NOW}}, {{KIMI_WORK_DIR}} 等
4. 工具名称大小写敏感
5. 子 Agent path 相对于 resources/agents 目录
